Plugin Levels

Levels of cloud integration

We support three levels of cloud:

1. Cloud Environment

Your cloud environment can expand to the Cloud Unit as an extra avaliability zone. It is an easy solution to comply with regulatory requirements or to bring your cloud environment to a specific geographical location. The cloud provider charges additional fees for running their full software stack on the Cloud Unit. But it is still less expensive than similar on-premise or co-location solutions.

  • AWS Outpost
  • Azure HCI

2. Cloud Services

The Cloud Unit can effortlessly be plugged into a range of Cloud services. That's it, you have just saved on cloud expenses and have become the greenest IT company (with an social impact).

  • Azure Functions
  • AWS Lambda
  • Kafka

Use case: Azure Service Bus

The Azure Service Bus Queue accumulates messages transmitted from various cloud components. In the Cloud Unit, hundreds of code instances are standing by to immediately process any new messages appearing in the Service Bus Queue. The local storage in the Cloud Unit is leveraged to improving inter-process latency and reducing network transit.

3. Software Stack

Imagine you can deploy your software stack effortlessly at any physical location. Ideal for low latency applications like IoT, 5G, local AI and Industry 4.0.

We deploy and maintain the physical Cloud Unit, so you can focus on the software. Use our API or Terraform plugin to deploy your favorite OS. We recomment using configuration management tools like Puppet, Chef, Ansible or SaltStack to deploy your software stack.

  • Puppet
  • Ansible
  • SaltStack
  • Terraform

During your journey, we love to advice you on how to utilize 100% of the Cloud Unit resources.
