Plugin Levels

Levels of cloud integration

We support three levels of integration with each its own cons and pros:

1. Full Cloud integration

Full Cloud integration allows you to deploy cloud services directly into the Micro Data Center. This solution is more expensive than the other solutions we provide because the cloud provider charges additional fees for running their cloud software outside their infrastructure. But still less expensive than similar on-premise or co-location solutions. Full cloud Integration is suitable if you must (by law) support the public cloud software locally.

  • AWS Outpost
  • Azure HCI

Do you have regulatory constrains and insist on using AWS outpost, Azure HCI or similar solutions? Then we can discuss how to integrate the cloud servers directly into the Micro Data Center. The benefit is that you can reuse the heat and do not need to build an expensive server room.

2. Integration with cloud service (FaaS/PaaS)

A Event Driver Architecture like AWS lambda, Azure Functions or Apache Kafka are very easy to connect with the Micro Data Center. Instead of running your logic in containers or FaaS runtime instances you deploy them into a Runtime instance in the Micro Data Center. That's it, you have just saved on compute expenses and have become the greenest IT company (with an social impact).

  • Azure Functions
  • AWS Lambda
  • Kafka

Use case: Azure Service Bus

The Azure Service Bus queue accumulates messages transmitted from various FaaS components. In the Micro Data Center, hundreds of code instances can be operated, which are effortlessly deployable via our API. These instances are designed to immediately process any new messages appearing in the Service Bus queue.

You could deploy code instances that require a high amount of compute resources or thousands of instances that process small compute jobs. The result can be stored locally into the Micro Data Center, effectively improving inter-process latency and reducing network transit.

3. You manage the full technology stack (IaaS)

You can use our API or Terraform plugin to (re-)deploy your favorite OS onto the servers. We recomment using configuration management tools like Puppet, Chef, Ansible or SaltStack to deploy software solutions.

  • Puppet
  • Ansible
  • SaltStack
  • Terraform

During your journey, we love to advice you on how to utilize 100% of the Micro Data Center resources.
