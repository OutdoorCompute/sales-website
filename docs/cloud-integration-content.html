Seamless Integration

Different levels integration with each its own cons and pros:

- No code modification (e.g., AWS Outpost or Azure HCI): These solutions are very expensive and are suitable if you must (by law) support the public cloud software on premise. In the Cloud you can deploy VMs and most other services directly into you own geographical area.

Do you have regulatory constrains and insist on using AWS outpost, Azure HCI or similar solutions? Then we can discuss how to integrate the cloud servers directly into the Micro Data Center. The benefit is that you can reuse the heat and do not need to build an expensive server room.

Do you have regulatory constrains but you maintain you own infrastructure or want to integrate with the cloud? Then the options below are valid.

- Integrate minimal code modification (Event Driver Architecture): A Event Driver Architecture like AWS lambda, Azure Functions or Apache Kafka are very easy to connect with the Micro Data Center. Instead of running your logic in containers or FaaS runtime instances you deploy them into a Runtime instance in the Micro Data Center. That's it, you have just saved on compute expenses and have become the greenest IT company (with an social impact).

- Fully based on the uDC: Using our API you can deploy a vanilla Operating System on one or more Micro Data Center server. At this point you can use SSH to log into the server and have full control. Each Compute flavor Micro Data Center has 16 server, each one has a 16 core CPUs which are in general 30% faster than the 64 er 128 core CPU's found in traditional data centers.

During your journey, we love to advice you on how to utilize 100% of the Micro Data Center resources.

Event Driven Architecture as a Service

Function as a Service (FaaS)

FaaS, or Function as a Service, is a cloud computing service that allows users to execute code in response to events without the complexity of building and maintaining the infrastructure typically associated with developing and launching an app.

Processes within a FaaS environment are activated in response to specific events, such as an HTTP request targeting a designated endpoint, or by various cloud components that are configured to initiate triggers within the cloud's FaaS framework.

The Micro Data Center can hook into those triggers and become fully integrated into the cloud's FaaS framework.

Examples of FaaS services are: AWS Lambda, Cloudflare Workers, Azure, DigitalOcean, Google Cloud, Oracle Cloud and Netlify functions.

Use case: Azure Service Bus

The Azure Service Bus queue accumulates messages transmitted from various FaaS components. In the Micro Data Center, hundreds of code instances can be operated, which are effortlessly deployable via our API. These instances are designed to immediately process any new messages appearing in the Service Bus queue.

You could deploy code instances that require a high amount of compute resources or thousands of instances that process small compute jobs. The result can be stored locally into the Micro Data Center, effectively improving inter-process latency and reducing network transit.

Platform as a Service (PaaS)

Cloud platforms offer containers that can be directly used without maintaining the underlying systems (e.g. Kubernetes). In most cases the name "Kubernetes" is included into the PaaS service name: Elastic Kubernetes Service (EKS), Azure Kubernetes Service (AKS), IBM Cloud Kubernetes Service etc.

Code that currently runs in a container (e.g., Kubernetes, Docker, LXC) can also run in our Micro Data Center. Your base workload could run in the Micro Data Center, while scaling up into the cloud if necessary. Leveraging the cost-effective compute of the Micro Data Center and the scalability of the Cloud.

Event Driven Architecture Self-hosted

A self-hosted Event Driven Architecture (EDA) is similar to FaaS or PaaS. The difference is that with FaaS or Paas you are not responsible for maintaining and scaling the underlying software.

How to use the Micro Data Center when using self-hosted solutions?

  1. You can deploy your logic into it's own Micro Data Center instance, effectivily using our FaaS/FaaS service.
  2. Or run your software stack on a bare Micro Data Center server. You can use our API or Terraform plugin to re-deploy an OS onto the servers.

Examples of event drive architecture are: RabbitMQ, Apache Pulsar, Amazon Kinesis, Google Cloud Pub/Sub, Azure Event Hubs, EventStoreDB and ActiveMQ.
